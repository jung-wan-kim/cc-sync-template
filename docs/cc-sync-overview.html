Problem

Why You Need This

01

Settings Loss

~/.claude/ only exists locally. An OS reinstall or disk failure means hundreds of hours of custom settings vanish irreversibly.

02

Device Fragmentation

Laptop, desktop, office PC. Skills, hooks, and rules evolve independently on each device, fragmenting your workflow.

03

Manual Sync Limits

Miss one file and things break. No way to track which device has the latest version. Syncing memory makes it even harder.

Commands

Usage

/cc-sync
~/.claude/ → git repository
Export local settings to the Git repo. Auto-commits and pushes in one step.
./sync.sh --auto
/cc-apply
git repository → ~/.claude/
Apply repo settings to local machine. Use for new device setup or restore.
git pull && ./apply.sh
import-sync
git → local memory-bank
Import memory-bank facts accumulated on other devices into local DB.
./import-sync.sh
/loop
automated schedule
Auto-sync daily. Runs for 7 days while the session is active.
/loop 1d /cc-sync
